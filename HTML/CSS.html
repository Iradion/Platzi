Las tres formas de hacerlo

Link referenciado

Creando un documento especifico para el estilo de nuestras páginas y llamándolo desde la cabecera(<head>).

Etiqueta

Creándolo con <style> dentro del mismo codigo HTML de la página.

Atributo

Dándole un atributo al elemento que queremos editar.

De las tres opciones la mejor práctica es usar la de link referenciado, solo en casos especificos usaremos las otras dos opciones.

Estructura básica de CSS.

Selector:

Como su nombre lo indica, sirve para elegir el elemento a editar. Se pueden crear clases, ID y otro tipo de especificaciones para hacer selecciones más especificas.

Propiedad:

Que tipo de edición se va a hacer.

Valor:

Valor de la edición en cuestión.

¿Qué es BEM?

En palabras simples, una forma ordenada de nombrar a nuestras clases que al mismo tiempo nos beneficia en nuestro codigo.

Visto desde atrás:

 HTML:
          <button class="button">
           Botón Normal  
           </button> <button
          class="button button--state-success">
           Botón éxito
          </button> <button class="button button--state-danger">
           Botón peligro <
          button>
        
 CSS:
          .button {
	display: inline-block;
	border-radius: 3px;
	padding: 7px 12px;
	border: 1px solid #D5D5D5;
	background-image: linear-gradient(#EEE, #DDD);
	font: 700 13px/18px Helvetica, arial;
}
.button--state-success {
	color: #FFF;
	background: #569E3D linear-gradient(#79D858, #569E3D) repeat-x;
	border-color: #4A993E;
}
.button--state-danger {
	color: #900;
}

Resultado:

El objetivo final de BEM es dividir la interfaz de usuario en bloques independientes para crear componentes escalables y reutilizables. Propone un estilo descriptivo para nombrar cada una de las clases a utilizar, permitiendo así crear un estructura de código consistente.
